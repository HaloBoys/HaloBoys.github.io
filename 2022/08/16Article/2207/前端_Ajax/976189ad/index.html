网络传输协议

常见协议

  1. HTTP HTTPS 超文本传输协议
  2. FTP 文件传输协议
  3. SMTP 简单邮件传输协议

HTTP协议

HTTP协议是超文本传输协议，网站是基于HTTP协议的，例如网站的图片、CSS、JS等都是基于HTTP协议进行传输的。

HTTP协议主要由请求和响应构成（HTTP协议是由从客户机到服务器的请求(Request)和从服务器到客户机的响应(Response)进行了约束和规范）

常用请求方法

POST

特性

  1. GET请求是放在url中，POST则放在body中

GET

特性

  1. GET 比POST更不安全，因为参数直接暴露在URL上，所以不能用来传递敏感信息。
  2. GET 请求在URL中传送的参数是有长度限制的，而POST没有

PUT

DELETE

状态码

常见

  1. 200代表成功
  2. 304文档未修改
  3. 403没有权限
  4. 404未找到
  5. 500服务器错误
状态码 含义
100~199 表示成功接收请求，要求客户端继续提交下一次请求才能完成整个处理过程
200~299 表示成功接收请求并已完成整个处理过程
300~399 为完成请求，客户需进一步细化请求。例如，请求的资源已经移动一个新地址
400~499 客户端的请求有错误
500~599 服务器端出现错误

请求/请求报文

规范格式：请求行、请求头、请求主体

简单概括：请求报文包括：请求行、请求头、请求主体，主要有送的方法，浏览器的信息，和发送的数据。

请求行

由 请求方式、请求URL、协议版本 构成

请求头

Host：localhost请求的主机
Cache-Control：max-age=0控制缓存
Accept：/ 接受的文档MIME类型
User-Agent：很重要
Referer：从哪个URL跳转过来的
Accept-Encoding：可接受的压缩格式

请求主体

传递给服务端的数据

注：以post形式提交表单的时候，请求头里会设置 Content-Type: application/x-www-form-urlencoded，以get形式当不需要。

响应/响应报文

规范格式：状态行、响应头、响应主体

简单概括：响应报文包括：状态行、响应头、响应主体，主要是服务器的信息浏览器中直接可以看到的内容。

状态行

由协议版本号、状态码和状态信息构成

响应头

Date：响应时间
Server：服务器信息
Content-Length：响应主体长度
Content-Type：响应资源的MIME类型

响应主体

服务端返回给客户端的内容

XML (了解)

  • XML是一种标记语言，很类似 HTML，其宗旨是用来传输数据，具有自我描述性（固定的格式的数据）
  • 虽然可以描述和传输复杂数据，但是其解析过于复杂并且体积较大，实际开发很少使用

基本语法

  1. 必须有一个根元素
  2. 不可有空格、不可以数字或 . 开头、大小写敏感
  3. 不可交叉嵌套
  4. 属性双引号（浏览器自动修正成双引号了）
  5. 特殊符号要使用实体
  6. 注释和HTML一样

解析

解析步骤类似使用 JS 获取 DOM 元素。

JSON

JSON (JavaScript Object Notation) 是一种语法，用来序列化对象、数组、数值、字符串、布尔值和 null，是一种轻量级的文本数据交换格式。

特性

  1. JSON 的载体是字符串
  2. JSON 数据的属性名、属性值，必须使用双引号包裹。
  3. JSON 对象键值对之间使用逗号（,）分隔。

方法

JSON.parse()

JSON.stringify()

AJAX 编程

Ajax 即 Asynchronous Javascript And XML，AJAX 不是一门的新的语言，而是对现有持术的综合利用。本质是在HTTP协议的基础上以异步的方式与服务器进行通信。

XMLHttpRequest

XMLHttpRequest 是浏览器内建对象，用于在后台与服务器通信(交换数据) ，由此我们便可实现对网页的部分更新，而不是刷新整个页面。

GET 和 POST请求区别（面试）

  1. GET 没有请求主体，使用 xhr.send(null)
  2. GET 可以通过在请求 URL上添加请求参数
  3. GET 大小限制约 4K，POST则没有限制
  4. POST 可以通过 xhr.send(‘xxx=xxx&xxx=xxx’)
  5. POST 发送数据时需要设置请求头 xhr.setRequestHeader("Content-type","application/x-www-form-urlencoded");

API

  1. xhr.open() 规定请求的类型、URL 以及是否异步处理请求
  2. xhr.setRequestHeader() 设置请求头
  3. xhr.send() 发送请求主体get方式使用xhr.send(null)
  4. xhr.onreadystatechange = function () {} 监听响应状态
    1. xhr.readyState = 0 时，UNSENT open尚未调用
    2. xhr.readyState = 1 时，OPENED open已调用
    3. xhr.readyState = 2 时，HEADERS_RECEIVED 接收到头信息
    4. xhr.readyState = 3 时，LOADING 接收到响应主体
    5. xhr.readyState = 4 时，DONE 响应完成
  5. xhr.status表示响应码，如200
  6. xhr.statusText表示响应信息，如OK
  7. xhr.getAllResponseHeaders() 获取全部响应头信息
  8. xhr.getResponseHeader(‘key’) 获取指定头信息
  9. xhr.responseText、xhr.responseXML都表示响应主体

补充：onreadystatechange 事件 是 onload 事件的兼容性方案。
