定义组件

使用 Vue.extend(options) 创建
其中 options 和 new Vue(options) 时传入的那个 options 几乎一样，但也有点区别:

  1. el 不要写 ——— 最终所有的组件都要经过一个vm的管理，由vm中的el决定服务哪个容器。
  2. data 必须写成函数 ———— 避免组件被复用时，数据存在引用关系。

补充：使用 template 可以配置组件结构。

注册组件

  1. 局部注册：new Vue 的时候传入 components 选项
  2. 全局注册：Vue.component('组件名',组件)

注意事项

组件命名

一个单词组成：

  1. 第一种写法(首字母小写)：school
  2. 第二种写法(首字母大写)：School

多个单词组成：

  1. 第一种写法(kebab-case命名)：my-school
  2. 第二种写法(CamelCase命名)：MySchool (需要Vue脚手架支持)

补充：

  1. 组件名尽可能回避HTML中已有的元素名称，例如：h2、H2都不行。
  2. 可以使用name配置项指定组件在开发者工具中呈现的名字。

组件标签

第一种写法：<组件名></组件名>

第二种写法：<组件名/>

备注：不用使用脚手架时，<组件名/>单标签会导致后续组件不能渲染。

组件嵌套

单文件组件

文件扩展名为 .vue 的 就是 single-file components (单文件组件)

VueComponent

  1. 组件实例对象(vc)，本质是一个名为 VueComponent 的构造函数，是 Vue.extend 生成的。
  2. 只需要写 <组件名/> 或 <组件名></组件名> ，Vue 解析时会帮我们创建 对应组件的实例对象，即Vue帮我们执行的：new VueComponent(options)。
  3. 关于this指向：
    1. 组件配置中：data函数、methods中的函数、watch中的函数、computed中的函数 它们的 this 均是 VueComponent实例对象。
    2. new Vue(options)配置中：data函数、methods中的函数、watch中的函数、computed中的函数 它们的 this 均是 Vue实例对象。
  4. 特别注意：每次调用Vue.extend，返回的都是一个全新的 VueComponent ！
  5. 一个重要的内置关系：VueComponent.prototype.__proto__ === Vue.prototype，为什么要有这个关系：让组件实例对象（vc）可以访问到 Vue原型上的属性、方法。

render 函数

因为 vue.runtime.xxx.js 没有模板解析器，所以不能使用 template 配置项，需要使用 render 函数接收到的 createElement 函数去指定具体内容。

关于不同版本的Vue：

vue.js 与 vue.runtime.xxx.js 的区别：

  1. vue.js 是完整版的 Vue，包含：核心功能+模板解析器。
  2. vue.runtime.xxx.js 是运行版的 Vue，只包含：核心功能；没有模板解析器。

ref 属性

ref 属性被用来给元素或子组件注册引用信息，引用信息将会注册在父组件的 $refs 对象上。（id的替代者）
ref 属性应用在html标签上获取的是真实DOM元素，应用在组件标签上是组件实例对象

使用：

html标签：<h1 ref="xxx">.....</h1>
组件标签：<组件名 ref="xxx"></组件名>

获取：

this.$refs.xxx

特点

props 是只读的

props 是只读的，Vue底层会监测你对 props 的修改，如果进行了修改，就会发出警告。(如果对对象中的属性修改，vue不会检测到并发出警告，不过不推荐这么做。)
若业务需求确实需要修改，那么请复制 props的内容到 data中，然后去修改data中的数据。

插槽

插槽作用：让父组件可以向子组件指定位置插入 html 结构，也是一种组件间通信的方式，适用于父组件向子组件传递数据。

默认插槽

应用场景

业务逻辑简单，直接在组件标签内传递插槽中的内容

语法

  1. 在子组件中使用：<slot>[default content...]</slot> 标签定义插槽
  2. 在父组件中使用 <子组件名>...填充内容...</子组件名> 的方式填充插槽

具名插槽

应用场景

需要向插槽中传递多段 html结构 时

语法

  1. 在子组件中使用：<slot name= "名称">[default content...]</slot> 标签定义插槽
  2. 在父组件中使用 <template v-slot:名称>...</template> 的方式填充插槽
    1. 也可以使用 <标签名 slot = "名称">...</标签名> 的方式填充插槽
    2. v-slot 指令只能在 template 标签中使用
  3. 具名插槽的缩写: v-slot 可以缩写，即把参数之前的所有内容 (v-slot:) 替换为字符 #。例如 v-slot:header 可以被重写为 #header

作用域插槽

应用场景

数据在组件的自身，但根据数据生成的结构需要组件的使用者来决定。

语法

  1. 在子组件中使用：<slot :数据名=数据源>[default content...]</slot> 标签定义插槽
  2. 在父组件中使用 <template v-slot:default = "数据">...</template> 的方式填充插槽
    1. 接受到的 数据 是一个对象,对象中保存着传递过来的数据
    2. 可以使用 es6 解构赋值 的方式接收数据
    3. 除了可以使用 v-slot:default = "数据" 指令语法接受数据以外,以下三种写法也奏效,不过不推荐
      1. scope=”数据”
      2. slot-scope=”数据”
      3. v-slot =”数据”

全局事件总线（GlobalEventBus）

适用于任意组件间通信。（本质是自定义事件）

实现原理：

  1. 在所有组件实例对象（vc）都能访问到的地方：Vue 的原型上添加一个变量：$bus（因为有一个重要的内置关系：VueComponent.prototype.__proto__ === Vue.prototype）
  2. 接收数据的一方给 $bus 绑定自定义事件，事件的回调留在接收数据的组件自身
  3. 提供数据的一方使用 $bus.$emit('xxxx',数据) 的方式传递数据

$attrs 与 $listeners

应用场景：封装第三方组件

vm.$attrs

$attrs 属于组件的一个属性，可以获取到父组件传递过来的props数据 (如果子组件通过props接收过的属性，在$attrs属性当中是获取不到的)

在子组件中可以使用：v-bind= "$attrs" 将父组件穿过来的属性绑定在子组件上（注意此处不能使用 v-bind 的简写形式）

vm.$listeners

$listeners 属于组件的一个属性，可以获取父组件传递过来的自定义事件

在子组件中可以使用：v-on="$listeners" 将父组件穿过来的自定义事件绑定在子组件上（注意此处不能使用 v-on 的简写形式）

当组件和混入对象含有同名选项时，这些选项将以恰当的方式进行合并

nextTick

nextTick：在下次DOM更新循环结束之后执行延迟回调。在修改数据之后立即使用这个方法，获取更新后的DOM。
应用场景：当改变数据后，要基于更新后的新 DOM 进行某些操作时，要在 nextTick 所指定的回调函数中执行。
