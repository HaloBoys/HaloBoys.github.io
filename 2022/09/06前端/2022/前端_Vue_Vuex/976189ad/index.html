Vuex

发表于2022-09-06|更新于2023-07-14|前端Vue
|字数总计:1.8k|阅读时长:7分钟

介绍

Vuex官方文档: https://vuex.vuejs.org/

概念

在 Vue 中实现集中式状态（数据）管理的一个 Vue插件。
对 vue应用中多个组件的共享状态进行集中式的管理（读/写），也是一种组件间通信的方式，且适用于任意组件间通信。

多组件共享数据对比

全局事件总线实现

使用全局事件总线可以勉强实现,但是随着组件增多,代码会变得越来越难以维护.

vuex 实现

Vuex 工作原理

应用场景

多个组件共享同一份状态，组件的共享状态抽取出来，以一个全局单例模式管理

  1. 多个组件依赖于同一状态
  2. 来自不同组件的行为需要变更同一状态

搭建 vuex 环境

安装

说明: vue2.x 版本需指定安装 vuex3 版本 npm i vuex@3（默认安装的是 vuex4 版本，只能在 vue3.x 中使用）

创建

创建文件：src/store/index.js

传入 store

在 main.js 中创建 vm 时传入 store 配置项

安装完成后，所有组件实例上都有 $store 属性

组件中读取 vuex 中的数据：$store.state.xxx
组件中修改 vuex 中的数据：$store.dispatch('action中的方法名',数据) 或 $store.commit('mutations中的方法名【方法名建议大写】',数据)【越过actions】

核心概念

State

Action

Action 中可以书写业务逻辑，但是不能修改 state

Mutation

Getter

当 state 中的数据需要经过加工后再使用时，可以使用 getters 加工。（可以理解为计算属性）。

在 store.js 中追加 getters 配置

在组件中使用：$store.getters.方法名称 来获取 getters 加工后的结果。(可以在任意组件中复用)

Module

目的：让代码更好维护，让多种数据分类更加明确。

四个 map方法的使用

使用前需要现在对应组件中引入: import {mapxxxx} from 'vuex'

mapState

mapState 用于帮助我们映射state中的数据为计算属性

mapGetters

mapGetters 用于帮助我们映射getters中的数据为计算属性

mapActions

mapActions 用于帮助我们生成与actions对话的方法，即：包含$store.dispatch(xxx)的函数

mapMutations

mapMutations 用于帮助我们生成与mutations对话的方法，即：包含$store.commit(xxx)的函数

TEMP

  1. import 导入文件时，会执行对应文件中的代码，import 语句比普通语句先执行。（无关先后顺序）
  2. 若没有网络请求或其他业务逻辑，组件中也可以越过 actions，即不写dispatch，直接编写commit
    1. mutations 接收的参数（通过 commit 提交的参数）建议使用大写（语义化）
  3. actions 中 context 上下文参数中的属性：
    1. dispatch：当业务逻辑很复杂的时候，可以借助 dispatch 参数将业务逻辑抛给下一个 actions
    2. commit
    3. getters
    4. rootGetters
    5. rootState
    6. state：state中的共享参数
  4. vue 开发者工具只检测 mutations 阶段中的数据变化
