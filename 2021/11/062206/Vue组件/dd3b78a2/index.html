vue组件

发表于2021-11-06|更新于2022-05-30|前端
|字数总计:2k|阅读时长:7分钟

组件系统让我们可以用独立可复用的小组件来构建大型应用，几乎任意类型的应用的界面都可以抽象为一个组件树

Feature

  • 组件中的 data 必须是一个函数，return 一个对象
  • 组件中的 this 表示当前组件对象的实例对象
  • 组件 template 标签中只能出现一个根元素

  • 工程化项目中，组件被封装好之后，彼此之间是相互独立的，不存在父子关系，在使用组件的时候，根据彼此的嵌套关系，形成父子、兄弟关系

  • 组件中的样式是全局生效的，需要在组件 style 标签上添加 scoped 属性避免污染全局样式，添加之后vue底层会为每个标签添加 v-data 属性

组件注册

  • 在组件中，通过 componets 注册的是私有子组件

  • 注册全局组件

    • main.js 文件中，通过 Vue.componet() 方法，注册全局组件

组件数据传递

props

父组件向子组件传递数据，在子组件中使用 props: ["attribute"] 定义该属性(该属性和 data 平级)，父组件中通过 v-bind:attribute = “val” 来传递值

props 的值是只读的,想要修改props 的值，可以把 props 的值copy一份到 data 中

props 的类型

props 的值可以为数组和对象类型，对象类型中可以对传递的值进行验证

  • 数组：props: [“attribute”]

    只能接收值，缺陷 ：如果外界没有传递初始值，这里就是 undefinde

  • 对象

一个组件默认可以拥有任意数量的 prop，任何值都可以传递给任何 prop
每次父级组件发生变更时，子组件中所有的 prop 都将会刷新为最新的值(同步)。这意味着你不应该在一个子组件内部改变 prop。如果你这样做了，Vue 会在浏览器的控制台中发出警告。

$emit

子组件中需要向父组件中传递数据，在子组件中使用 this.$emit('事件名称',子组件中要传递的数据) 方法传递数据，父组件中使用子组件在子组件标签中使用 @事件名称 = '方法' 的形式来接收数据

EventBus

兄弟组件数据共享方案，原理：创建一个中间模块，此模块向外共享一个Vue的实例对象，在数据的发送方，通过 实例对象.$emit(‘事件名称’，要发送的数据) 定义一个自定义事件，在数据的接收方在created 生命周期函数中通过 实例对象.$on(‘事件名称’，事件处理函数) 注册事件接受数据

实现详情

  1. 创建 eventBus.js模块，向外共享一个 Vue实例对象

  2. 数据发送方，通过 实例对象.$emit 定义事件,将要发送的数据传递给事件

  3. 数据接受方，通过 实例对象.on('事件名称'，事件处理函数) 接收数据

/deep/

当组件中使用第三方组件库，需要修改第三方组件的样式需求可以用 /deep/

Vue 中的 DOM 操作

vm.$ref

Vue 中 可以通过 ref 的形式获取DOM元素或组件引用

  1. DOM 元素引用操作

    1. 在 元素上绑定一个 ref = ‘属性’ 来为元素创建一个引用
    2. 在方法中通过 this.$refs.’属性’ 的方式来获取DOM元素

  2. 组件引用

    可以为组件绑定 ref 在父组件中可以直接获取或调用子组件中的属性或方法

    1. 为组件绑定 ref = ‘属性’ 来为组件创建一个引用
    2. 在父组件方法中通过 this.$refs.’属性’ 的方式来获取组件的引用

动态组件

动态组件使用 标签来进行组件占位，通过is 属性控制组件的是否展示

keep-alive

使用 keep-alive 标签包裹组件可以维持组件的状态，避免组件在切换过程中被销毁

keep-alive 生命周期函数

  • 当组件被缓存的时候，会自动触发组件的 deactivated 生命周期函数
  • 当组件被激活的时候，会自动触发组件的 activated 生命周期函数

keep-alive 属性 include & exclude

默认被 keep-alive 标签包裹的组件都会被缓存，可以使用 include & exclude 属性来有条件地缓存组件，多个组件之间用 英文逗号 分隔也可用正则表达式或一个数组来表示
include 与 exclude 不可同时使用

组件 name 属性

当组件设置了 name 属性之后，组件的名称（并非注册名称）就是 name 属性的值

Slot 插槽

当封装一个组件，不确定某个区域未来会被填充什么内容的时候，可以使用 slot 插槽来占位，每一个 slot 插槽都要有一个 name 属性，如果省略了 slot 的 name 属性，则会有一个默认的名称 ：name: default，在父组件中填充的内容默认会被放到这个插槽下

V-Slot

在封装组件的时候，如果指定了 slot 的 name 属性，则在父组件中插入元素的时候使用 的方式来使用插槽

  • 注意 v-slot 属性不能直接用在标签上
  • v-slot：简写形式是 #
  • 如果希望插槽有默认内容，可以在 //…default… 标签中填写默认内容,使用插槽的时候默认内容将会被替换

作用域插槽

在封装组件时，为预留的 slot 提供属性与对应的值，父组件中通过 v-slot:name = “属性” 的方式接收组件中的值，slot 不提供属性与对应的值默认是一个对象 {}
