损耗问题

目前我个人有个疑问：五金的材料损耗率在父级上统一都设置的 0.6%
如果在每个流程都损耗一部分，越往后，它的实际数量都是小于订单数量的，我比较好奇这个差异怎么解决

答：实际剩多少数量，就发多少数量

物料镭雕问题

场景描述：有一款通用物料A，用在BOM1和BOM2上，其中BOM1需要在厂内使用激光机打印客户logo，BOM2则不需要。目前直接把需要镭雕的描述写在A物料上，导致BOM2不需要镭雕也有这个描述，目前考虑将描述全部改到成品上。是否有其他的思路?

  1. 建2个料号，
  2. 只有一个料号的情况下，用辅助属性来区分
  3. 在成品描述中描述是否镭雕，在领料单中将成品描述打印出来

计划运算BOM包材还未确认的问题

先推BOM中的结构件和电子件，然后再推包材，一般包材确认的比较晚，像重复利用(返单)的BOM,里面的包材先不跑BOM,但是又没有确认，又不想删除的情况

方案1：包材和结构电子分开

方案2：删除包材跑MRP，包材确认再跑MRP

方案3: 物料维护计划标识，计算时可以把包材排除在外，又可以单独计算包材

目前老师提供了另一种可以不频繁删除BOM中包材的方案：
在物料上做两种标识（如果确认用此方案具体的细节我会去做），一种标识是跑BOM中结构件和电子料的，另一种标识是专门用于跑BOM中包材的，从而达到不删除包材也可以跑准需求

江门五金部：自制壳体，在系统中的流程制定

产品结构BOM的作用及规范

江门现在成立了五金部，职责之一就是部分产品的壳体由外购变成自制了
大致流程：

  1. 跟订单数量和单号绑定，下订单之后对应的壳体需要买原材料
  2. 原材料买回之后，需自己冲压
  3. 冲压完成之后外发喷粉
  4. 壳体成品入库，发给产线组装成最终成品

注意事项：

  1. 订单量大的情况如果自己做不过来会考虑直接外购
  2. 自制的情况下可以计算成本

方案1：把外壳做成委外的形式嵌套在原BOM中

假设 BOM1 中的外壳 A 需要原材料 B 去压铸成 C，再外发 A 出去喷粉，在系统中的流程：

  1. 把外壳 A 做成委外BOM的形式，里面有 B 和 C 并嵌套在BOM1中。（其中 A,B,C 为不同料号）
  2. 计划运算，会算出需要外发 A，采购 B，生产 C，
  3. 生产 C 领用 B，并入库 C（压铸过程）
  4. 外发 A 领用 B，并入库 A（外发喷粉）
    最后A和其他物料生产组装成 BOM1 的最终成品
