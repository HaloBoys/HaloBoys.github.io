前言

金蝶云产品

金蝶云是金蝶软件推出的云端服务平台，旨在为企业提供全方位的数字化转型解决方案。以下是金蝶云的主要产品：

  1. 金蝶云财务：提供财务核算、票据管理、资金管理、成本管理、报表分析等全面的财务管理解决方案。
  2. 金蝶云ERP：提供采购管理、销售管理、库存管理、生产管理、财务管理等企业管理全流程的ERP解决方案。
  3. 金蝶云HR：提供人事管理、薪酬管理、绩效管理、招聘管理、培训管理等全面的人力资源管理解决方案。

ERP 主要模块

模块 Description
财务会计模块 包括总账、应收应付、固定资产、现金管理等功能，可实现企业的财务核算和管理。
采购管理模块 包括采购订单、采购入库、采购发票等功能，可帮助企业实现采购流程的自动化管理。
销售管理模块 包括销售订单、销售出库、销售发票等功能，可帮助企业实现销售流程的自动化管理。
库存管理模块 包括库存调拨、库存盘点、库存预警等功能，可帮助企业实现库存管理的自动化。
生产管理模块 包括生产计划、生产领料、生产入库等功能，可帮助企业实现生产流程的自动化管理。
人力资源管理模块 包括招聘管理、员工档案、薪资管理等功能，可帮助企业实现人力资源管理的自动化。
项目管理模块 包括项目计划、项目预算、项目执行等功能，可帮助企业实现项目管理的自动化。

金蝶K/3 WISE

K/3 架构

数据库端

金蝶K/3系统采用大型网络数据库管理系统，支持大用户量的访问和海量的数据存储，支持主流大型数据库，如：MS SQL SERVER 2000/2005/2008,暂不支持DB2、Oracle、Sybase、mySQL等其他数据库。

中间层

负责业务逻辑运算（包括权限检查）,将客户端请求分解成不同的数据层操作，然后将处理结果返回给客户端。核心的业务处理都是通过中间层来实现的。中间层由各个模块系统的不同的业务组件构成，这些组件在MTS或者COM+环境中运行。

客户端

客户端只是用户的界面外壳，有GUI和WEB两种形式，不具有任何的复杂计算，它需要做的工作就是将中间层传入的业务对象数据放置在界面和控制用户的键盘鼠标操作。无论是用GUI还是用WEB最终完成的是同样工作，看到的是同样的结果，存储的是完全一致的数据。

中间层组件注册

同名同密码认证

在中间层服务器和K3客户的机器上建立相同用户名和密码的操作系统帐户，并且在客户端需要此用户登陆系统。

信任注册认证

不需要在中间层和客户端添加同名同密码的帐户，客户端使用GUEST用户访问服务器。

通过域控制器认证

中间层和客户加到相同的域里面，客户端和中间层之间的认证通过域控制器来完成。

信任方式

  1. 新建一个操作系统管理员用户，例如：新增 kingdee 用户（需加入管理员组）
  2. 注册中间层，选信任方式，输入用户：wilkeny\kingdee,计算机名\用户名格式或手工注册中间层
  3. 在进行中间层信任注册之后，依次打开【管理工具】->【本地安全策路】->【本地策路】->【安全选项】：
    1. 网络访问：本地账户的共享及安全模式更改为“仅来宾”：
    2. 帐户：使用空白密码的本地帐户只允许进行控制台登录更改为禁用：
    3. 账户：来宾账户状态设置为启用。
  4. 【本地策路】-> 用户权限分配，双击打开【拒绝从网络访问这台计算机】，将列表中的用户清空。
  5. 依次打开【管理工具】->【计算机管理】->【本地用户和组】->【用户】，设置GUEST用户密码为空，
  6. 依次打开【管理工具】->【组件服务】，弹出【组件服务】窗口中
    1. 依次展开【计算机】->【我的电脑】->【右键属性】->【COM安全】，启动和激活权限中Everyone的都允许：
    2. 依次展开【计算机】->【我的电脑】->【COM+应用程序】，选择所有的 ebo 开头的组件和 kdsvrmgr 组件，注意需要排除“EBOHRMS”，然后点击右键选择【属性】，点击【标识】，勾选“下列用户”，点击浏览选择管理员权限的用户并输入密码后单击【确定】：
    3. 依次展开【计算机】->【我的电脑】->【Distributed Transaction Coordinator】->【本地DTC】->【右键属性】->【安全】，勾选四个允许、不要求进行验证，

多语言部署规则

K/3从10.1开始支持多语言应用，但V11多语言版本之前，对繁体/英文只能局部的支持

中间层控制：

  1. 纯英文模式下的中间层：只能新建英文账套，系统预设的一些数据如科目等都是英文。
  2. 纯繁体模式下的中间层：只能新建繁体账套，系统预设的一些数据如科目等都是繁体。

客户端登录控制：

  1. 纯英文客户端，不能在登录界面进行语言切换，且只能通过纯英文中间层连接对应的英文账套。
  2. 纯繁体客户端，不能在登录界面进行语言切换，且只能通过纯繁体中间层连接对应的繁体账套。

账套备份常见问题

  1. 备份账套时提示备份失败，这种情况一般是备份文件名称或备份路径过长或者有中文字符引起的，需要改成英文短字符，
  2. 批量备份执行不成功，没有产生备份文件，可查看备份日志的记录的信息进行排查，
  3. 同一个时间点重复备份，别除原备份方案，重新新建备份方案再执行，
  4. 设置自动备份时间，第一次备份的时间 = 备份开始时间 + 备份间隔时间。
  5. 异地备份要求有异地备份文件夹添加 EVERYONE 完全控制权限，SQL服务的启动用户需要设置成 administrator

组件跟踪工具的使用

从10.2版本开始，其客户端的安装目录中附带了一个功能强大的调试工具 KDMAINDBG.EXE, 该工具可以跟踪 COM + 组件 的运行情况

该工具存放在K3软件安装路径的 K3ERP 文件夹下，其执行文件的名称是 kdmaindbg.exe, 启动后系统会提示您进入的是调试版，点击【确定】后，程序会自动打开K/3主控台的登陆窗口，同时也会开启一个组件跟踪窗口，实时记录当前K/3组件的调用情况，如果有组件出现异常，那么通过它非常方便定位问题并解决问题，另外该工具还跟踪了对象的执行时间，可以用来检测性能数据。当我们跟踪到有组件出现错误时，会出现一条红色显示的记录，表示该组件调用异常。

组件注册方法

中间层COM+组件需要在组件服务里面进行注册，依次打开【管理工具】-【组件服务】，弹出【组件服务】窗口。展开【组件服务】-【计算机】~【我的电脑】~【COM应用程序】，可以看到里面的COM+组件包，展开组件包可以看到每个组件包下包含的组件，组件和组件包的对应关系可以通过K3ERP\KDSYSTEM\KDCOM\MtsSpSetup.Lst文件查看。

COM+组件可以通过拖动dll文件到窗口里进行注册，也可以通过新增组件的方式进行注册。

分开部署的设置

分开部署顾名思义就是中间层服务器和数据库服务器分别安装在不同的服务器上，为了进一步提高金蝶K/3系统的性能与数据的安全性，经常要将用到分开部署的模式。

为了保证中间层和数据库服务器之间能够互相通信，需要做相同的设置如下：

  1. 添加应用程序服务器
    1. WIN2016 indows Server系统角色没有应用程序服务器，当分布式部署时，需要手动启动00I+网络访问，修改的注册表路径为：HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\COM3\RemoteAccessEnabled值修改为1即可，
  2. 配置COM+和DTC安全
  3. 添加host解析
  4. 关闭防火墙或者设置端口，中间层服务器上需要开放的端口有：
    1. TCP端口135，是远程过程调用服务(RPC)的固定端口，不能改变。与数据库服务器、Web服务器、客户端通讯时均需使用：
    2. TCP端口5159，K/3中间层加密服务的固定端口，不能改变，用于中间层与客户端、Web服务器通讯，中间层与数据库之间通讯不需此端口。
    3. DCOM 动态端口，DTC服务默认动态端口为 TCP1025~65534,可以更改。对于中间层不在防火墙内的环境可保持此默认值，中间层在防火墙内时可通过设置将动态端口改到较小范围，一般设置500个连续端口即可，例如 5000-5500。这些端口在K/3运行时并不处于侦听状态，只会动态调用和释放。

数据库设置

SQL SERVER网络配置：

在数据库服务器上，依次单击【开始】一【所有程序】→【Microsoft SQL Server】,【配置工具】，一【SQL Server配置管理器】，在弹出的“SQL Server Configuration Manager”窗口中单击展开【SQL Server网络配置】，单击【MSSQLSERVER的协议】，在右面的窗口中依次右击启用【Shared Memory】、【Named Piers】、【TcP/IP】。
